# MITO Blog

Flask 製の個人ブログアプリケーションです。単一の管理者が記事を投稿・管理し、閲覧者が記事を読む「1 人運用のブログ」を想定して作られています。Markdown での執筆に加え、画像・地図・YouTube の埋め込み、ジャンル／ハッシュタグによる分類・検索、ダークモード対応など、個人ブログに必要な機能を一通り備えています。

## スクリーンショット的な特徴

- **Coastal Dawn** をテーマにしたセルリアン基調のデザイン
- ライト／ダークモードの切り替え（選択は端末に記憶）

## 主な機能

機能を「誰でも使える閲覧者向け」と「ログインが必要な管理者向け」に分けて整理します。

### 閲覧者向けの機能

ログイン不要で、誰でも利用できる機能です。

| 機能 | 説明 |
|------|------|
| **記事一覧** | トップページに公開記事を新しい順で表示。ページ番号によるページ送りに対応（1 ページ 4 件） |
| **キーワード検索** | 記事タイトル・ハッシュタグ名に対する部分一致検索 |
| **ジャンル絞り込み** | ジャンルを選んで記事を絞り込み。ジャンル一覧ページはカテゴリごとのアコーディオン表示 |
| **ハッシュタグ絞り込み** | ジャンル選択中に、そのジャンル内で使われているタグでさらに絞り込み |
| **記事詳細** | Markdown 本文、自動生成の目次、画像（キャプション付き）、地図・YouTube 埋め込みを表示 |
| **関連記事** | 記事末尾に最大 4 件を表示。「同ジャンル×同タグ → 同タグ → 同ジャンル → 最新」の優先順位で選定 |
| **地図の埋め込み** | 本文中の場所が Google マップとしてその場で表示される |
| **YouTube の埋め込み** | サムネイルをタップすると再生が始まる軽量な「ファサード」方式 |
| **サイト統計** | トップページに総投稿数・ハッシュタグ数・最終更新日を表示 |
| **自己紹介／使い方ページ** | 管理者プロフィールとブログの使い方の案内 |
| **ダークモード** | ヘッダーのボタンで切り替え。初回はブラウザの設定に追従 |
| **レスポンシブ表示** | PC・スマートフォン双方に最適化されたレイアウト |

### 管理者向けの機能

隠しログインページからの認証後にのみ利用できる、記事管理のための機能です。

| 機能 | 説明 |
|------|------|
| **記事の投稿・編集・削除** | Markdown で本文を執筆。編集時は既存画像・キャプションの個別編集にも対応 |
| **Markdown 編集ツールバー** | H2/H3 見出し・太字・目次（`[toc]`）・箇条書き・地図・YouTube・画像挿入のワンタッチ挿入 |
| **画像アップロード** | 本文用画像を複数枚アップロード（`[imgN]` で本文に挿入）。各画像にキャプションを付与可能 |
| **画像の自動最適化** | アップロード画像は Pillow で長辺 1600px まで縮小・再圧縮して保存 |
| **サムネイル管理** | 専用サムネイル画像のアップロード（WebP 縮小）／プリセットからの選択／システム既定の 3 段階 |
| **ジャンル管理** | プリセットジャンルからの選択に加え、新しいジャンルの自作が可能 |
| **ハッシュタグ管理** | スペース・カンマ区切りで複数タグを入力。入力中にプレビュー表示。孤立タグは自動削除 |
| **公開／非公開設定** | 記事ごとに公開状態を切り替え。非公開記事は管理者のみ閲覧可能 |
| **マイページ** | 自分の投稿一覧（ページネーション）、使用ジャンル、総投稿数の確認とニックネーム変更 |

## ディレクトリ構成

```
.
├── app.py                 # アプリ生成のエントリーポイント（create_app）
├── config.py              # 環境変数の読み込み
├── constants.py           # ジャンル定義（唯一の情報源）
├── extensions.py          # Flask 拡張機能インスタンスの置き場
├── models.py              # DB テーブル定義（Post / Hashtag / User）
├── rendering.py           # 本文（Markdown + 独自タグ）→ HTML 変換
├── init_db.py             # SQLite 向けの DB 初期化スクリプト
├── views/
│   ├── auth.py            # ログイン・ログアウト
│   ├── blog.py            # 一般公開ページ（一覧・詳細・ジャンル）
│   └── admin.py           # 管理者ページ（投稿・編集・削除・マイページ）
├── templates/             # Jinja2 テンプレート
├── static/                # CSS / JS / 画像 / favicon
├── migrations/            # Alembic マイグレーション
├── docker-compose.yml     # ローカル開発用 PostgreSQL
├── requirements.txt       # 依存パッケージ（PostgreSQL 用）
├── requirements-pythonanywhere.txt  # SQLite 運用向けの依存
└── deploy_pythonanywhere.md         # PythonAnywhere デプロイ手順
```

## セットアップ

### 環境変数

プロジェクト直下に `.env` ファイルを作成し、以下を設定します（`.env` は Git 管理外です）。

| 変数名 | 必須 | 説明 |
|--------|------|------|
| `SECRET_KEY` | 本番で必須 | セッション・CSRF トークンの署名に使う秘密鍵 |
| `ADMIN_USERNAME` | ✅ | 管理者ログインのユーザー名 |
| `ADMIN_PASSWORD` | ✅ | 管理者パスワード（平文・ハッシュ済みのどちらでも可） |
| `ADMIN_LOGIN_PATH` | ✅ | ログインページの URL パス（推測されにくい文字列にする） |
| `ADMIN_GATE_KEY` | ✅ | ログインページを表示するための合言葉 |
| `FLASK_ENV` | 任意 | `production` を指定すると本番モード（SQLite 運用時は必須） |
| `USE_SQLITE` | 任意 | `1` で SQLite を使用 |
| `DATABASE_URL` | 任意 | 明示的な DB 接続 URL（最優先で使われる） |
| `POSTGRES_USER` / `POSTGRES_PASSWORD` / `POSTGRES_DB` | 任意 | ローカル PostgreSQL 接続用 |

ランダムな文字列は次のように生成できます。

```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

### ローカル開発（PostgreSQL / Docker）

```bash
# 1. 仮想環境の作成と有効化
python -m venv venv
source venv/bin/activate        # Windows は venv\Scripts\activate

# 2. 依存パッケージのインストール
pip install -r requirements.txt

# 3. PostgreSQL コンテナの起動（.env の POSTGRES_* を参照）
docker compose up -d

# 4. マイグレーションの適用
flask db upgrade

# 5. 開発サーバーの起動
python app.py
```

`DATABASE_URL` も `USE_SQLITE` も設定していない場合、`localhost:15432` の PostgreSQL（docker-compose 側）へ自動接続します。

### SQLite で手軽に動かす

DB サーバーを立てずにファイル 1 つで動かす場合は、`.env` に `USE_SQLITE=1` と `FLASK_ENV=production` を設定し、初期化スクリプトを実行します。

```bash
pip install -r requirements-pythonanywhere.txt
python init_db.py     # テーブル作成 + 管理者ユーザー作成
python app.py
```

`init_db.py` はテーブルと管理者ユーザーをまとめて作成し、Alembic の履歴も最新へスタンプするため、後からマイグレーション運用へ移行しても整合が保たれます。何度実行しても安全です。

## データベースのマイグレーション

`models.py` を変更した後は、以下でスキーマを更新します。

```bash
flask db migrate -m "変更内容の説明"   # 差分から新しいマイグレーションを生成
flask db upgrade                       # 実際の DB に適用
```

## デプロイ

PythonAnywhere 無料枠（SQLite・GitHub 経由）への詳細なデプロイ手順・更新運用・トラブルシューティングは [`deploy_pythonanywhere.md`](./deploy_pythonanywhere.md) にまとめています。

概略は次の通りです。

1. GitHub にコードを push
2. サーバー上で `git clone` → 仮想環境作成 → `pip install`
3. サーバー上で `.env` を作成
4. `python init_db.py` で DB を初期化
5. Web タブで WSGI・仮想環境・静的ファイルを設定して Reload

コード更新後は「PC で修正 → push → サーバーで `git pull` → **Reload**」が基本の流れです。

## 設計上のポイント

- **Application Factory パターン** — `create_app()` でアプリを組み立て、テスト時の差し替えや循環インポートの回避を容易にしています。
- **多層的なログイン防御** — ①秘密の URL、②ゲートキー（合言葉 Cookie）、③ユーザー名、④パスワードの 4 要素が揃わない限り管理画面に到達できません。未認証で管理ページへアクセスすると 404 を返し、存在自体を隠します。
- **ブルートフォース対策** — 連続ログイン失敗でセッション単位のロックアウトが働きます。
- **アップロードの多層検証** — 拡張子チェックに加え、ファイル先頭バイトから MIME タイプを判定して拡張子偽装を検出します。
- **画像・DB 操作のアトミック性** — DB の commit が成功した後にのみ旧ファイルを物理削除し、失敗時は新規保存ファイルを掃除することで、DB とファイルの不整合を防ぎます。
- **本文 HTML の事前生成** — Markdown＋独自タグの変換結果を投稿・編集時に `body_html` / `toc_html` へキャッシュし、閲覧のたびの再変換を無くしています。導入前の既存記事は初回閲覧時に遅延生成します。
- **ジャンル定義の一元管理** — ジャンルは `constants.py` の `GENRE_GROUPS` を唯一の情報源とし、一覧ページやフォームの選択肢を自動生成します。
- **N+1 対策とインデックス** — ハッシュタグを `selectin` ロード、主要な検索・並べ替えカラムにインデックス（複合含む）を付与しています。
