# MITO Blog

Flask 製の個人ブログアプリケーションです。管理者（ブログ運営者）が記事を投稿・管理し、閲覧者が記事を読む——という「1 人の管理者 + 複数の閲覧者」を想定した構成になっています。Markdown ベースの記事作成に加えて、地図・YouTube の埋め込み、ハッシュタグ、ダークモードなどに対応しています。

本番（PostgreSQL）と無料ホスティング（SQLite）の両方で動作するよう設計されており、Docker Compose によるローカル開発と PythonAnywhere へのデプロイ手順が用意されています。

## 主な機能

機能は「ログイン不要で誰でも使える閲覧者向け機能」と「管理者ログインが必要な管理機能」に分かれています。

### 閲覧者向けの機能

ログイン不要で利用できる、記事を読むための機能です。

- **記事一覧（トップページ）** — 公開記事を新しい順に表示。1 ページ 4 件のサーバーサイドページネーションに対応します。
- **キーワード検索** — タイトルおよびハッシュタグ名に対する部分一致検索。PostgreSQL では `pg_trgm` の GIN トリグラム索引により、部分一致でも索引が効くよう最適化されています。
- **ジャンル絞り込み** — ジャンル単位で記事をフィルタリング。ジャンル一覧ページはカテゴリ別のアコーディオン表示になっています。
- **ハッシュタグ絞り込み** — ジャンル選択中に、そのジャンル内で使われているタグでさらに細かく絞り込めます。
- **記事詳細ページ** — Markdown で書かれた本文を整形表示。見出しからの自動目次（TOC）、画像とキャプション、Google マップ埋め込み、YouTube 埋め込み（サムネイルをタップして再生するファサード形式）を表示します。
- **関連記事の表示** — 記事末尾に、同ジャンル・同タグを基準とした関連記事を最大 4 件表示します。
- **サイト統計** — トップページに総投稿数・ハッシュタグ数・最終更新日を表示します。
- **自己紹介ページ / 使い方ページ** — 管理者の紹介と、ブログの使い方の案内。
- **ダークモード** — ライト / ダークをワンタップで切り替え。選択内容はブラウザに保存され、次回訪問時にも維持されます。OS のカラースキーム設定にも追従します。
- **レスポンシブ対応** — PC・スマートフォンいずれの画面幅にも最適化されています。

### 管理者向けの機能

秘密の URL からログインした管理者だけが利用できる、記事を作成・管理するための機能です。

- **記事の投稿・編集・削除** — 記事の作成、既存記事の更新、削除に対応します。
- **Markdown エディタ** — 見出し（H2 / H3）、太字、目次（`[toc]`）、箇条書きリスト、地図、YouTube を挿入できるツールバー付き。スマホでは編集ツールバーがキーボード直上に固定され、カーソルが隠れないよう自動スクロールします。
- **画像アップロード** — 本文中に `[imgN]` で挿入する画像を複数枚アップロード可能。各画像にキャプションを設定できます。アップロード時に Pillow で自動的にリサイズ・再圧縮されます。
- **サムネイル管理** — 記事一覧で表示されるサムネイルを、専用画像のアップロード・プリセットからの選択・システム共通デフォルトの 3 段階の優先順位で設定できます。
- **地図・YouTube 埋め込み** — モーダルから場所名や URL を入力し、プレビューを確認しながら本文に挿入できます。
- **ハッシュタグ付与** — スペース・カンマ区切りで複数タグを入力（入力中にリアルタイムプレビュー）。使われなくなった孤立タグは自動的に削除されます。
- **ジャンル管理** — プリセットのジャンルから選択、または新規ジャンルを作成できます。
- **公開 / 非公開の切り替え** — 記事ごとに公開状態をトグル。非公開記事は管理者本人のみ閲覧できます。
- **マイページ** — 自分の投稿一覧（ページネーション）、総投稿数、使用ジャンルの確認、ニックネームの変更。

## アーキテクチャ

- **Application Factory パターン** — `app.py` の `create_app()` でアプリを組み立てます。テスト時に設定を差し替えやすく、循環インポートを避けられます。
- **Blueprint による機能分割** — ルートを役割ごとに 3 つの Blueprint に分けています。
  - `views/auth.py` — ログイン・ログアウト
  - `views/blog.py` — 一般公開ページ（一覧・詳細・ジャンル・自己紹介・使い方）
  - `views/admin.py` — 管理者専用ページ（投稿・編集・削除・マイページ）
- **拡張機能の分離** — `extensions.py` に `db` / `login_manager` / `migrate` のインスタンスを置き、`app.py` と `models.py` の双方から参照することで循環インポートを回避しています。
- **本文レンダリングのキャッシュ** — Markdown と独自タグ（`[imgN]` / `[map:]` / `[youtube:]`）の HTML 変換は `rendering.py` に集約し、投稿・編集時に一度だけ実行して結果を `Post.body_html` / `Post.toc_html` に保存します。詳細表示は保存済み HTML を出力するだけで、アクセスのたびの再変換を避けています。
- **定数の一元管理** — ジャンルのグループ定義は `constants.py` の `GENRE_GROUPS` を唯一の情報源とし、フラットな一覧やジャンル一覧ページの表示はそこから派生させています。

### データモデル

- **User** — 管理者。ユーザー名・ハッシュ化パスワード・ニックネームを持ちます。
- **Post** — ブログ記事。タイトル・本文・ジャンル・画像・公開状態・レンダリング済み HTML キャッシュなどを持ちます。
- **Hashtag** — ハッシュタグ。`post_hashtags` 中間テーブルを介して Post と多対多で紐づきます。

## セットアップ（ローカル開発）

ローカルでは Docker Compose で立ち上げた PostgreSQL に接続する構成です。

1. **リポジトリを取得して依存をインストール**

   ```bash
   python -m venv venv
   source venv/bin/activate        # Windows は venv\Scripts\activate
   pip install -r requirements.txt
   ```

2. **`.env` を作成**（[環境変数](#環境変数)を参照）

3. **PostgreSQL を起動**

   ```bash
   docker compose up -d
   ```

   `docker-compose.yml` の設定により、ホストの `15432` 番ポートでコンテナの PostgreSQL に接続できます。

4. **データベースを初期化**（[マイグレーション](#データベースの初期化マイグレーション)を参照）

5. **アプリを起動**

   ```bash
   flask run
   # または
   python app.py
   ```

   デバッグモードで起動したい場合は `.env` に `FLASK_DEBUG=1` を設定します（本番判定時は無効化されます）。

## 環境変数

`.env` をプロジェクト直下に置きます。`config.py` が `config.py` 自身の絶対パスを基準に読み込むため、どこから起動しても確実に読み込まれます。

| キー | 必須 | 説明 |
|------|------|------|
| `ADMIN_USERNAME` | ○ | 管理者ログインのユーザー名 |
| `ADMIN_PASSWORD` | ○ | 管理者パスワード（平文・ハッシュ済みのどちらも可） |
| `ADMIN_LOGIN_PATH` | ○ | ログインページの URL パス（推測されにくいランダム文字列）。未設定だと起動時にエラー |
| `ADMIN_GATE_KEY` | ○ | ログインページを表示するための「合言葉」。未設定だとログインページは常に 404 |
| `SECRET_KEY` | 本番○ | セッション・CSRF 署名用の秘密鍵。本番では必須 |
| `FLASK_ENV` | — | `production` で本番モード（Secure Cookie・SECRET_KEY 必須化）。SQLite 運用時は明示設定を推奨 |
| `DATABASE_URL` | — | DB 接続 URL。設定時は最優先で使用 |
| `USE_SQLITE` | — | `1` で `instance/blog.db`（SQLite）を使用。無料ホスティング向け |
| `POSTGRES_USER` / `POSTGRES_PASSWORD` / `POSTGRES_DB` | — | ローカル PostgreSQL（Docker）用の接続情報 |
| `FLASK_DEBUG` | — | ローカル開発時に `1` でデバッグモード |

ランダム文字列の生成例:

```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

DB 接続の優先順位は `(1) DATABASE_URL` → `(2) USE_SQLITE=1` → `(3) ローカル PostgreSQL` です。

## データベースの初期化・マイグレーション

### マイグレーション運用（PostgreSQL 推奨）

```bash
flask db upgrade          # 既存のマイグレーションを適用
```

モデルを変更したときは差分を検出してマイグレーションを生成します。

```bash
flask db migrate -m "変更内容"
flask db upgrade
```

### 初期化スクリプト（SQLite 向け）

マイグレーションを使わない環境（PythonAnywhere など）では、テーブル作成と管理者ユーザー登録を一括で行う `init_db.py` を使います。

```bash
python init_db.py
```

`db.create_all()` でテーブルを作成し、Alembic の履歴を最新（head）にスタンプするため、後からマイグレーション運用へ移行しても不整合が起きません。何度実行しても安全です。

## デプロイ

PythonAnywhere 無料枠（SQLite 運用）への詳細なデプロイ・運用手順は [`deploy_pythonanywhere.md`](./deploy_pythonanywhere.md) にまとめています。GitHub 経由での配布、WSGI 設定、公開後のコード更新フローまで解説しています。

概要としては次の流れです。

1. GitHub からサーバーへ `git clone`
2. 仮想環境を作成し `requirements-pythonanywhere.txt` をインストール
3. サーバー上に `.env` を作成（`USE_SQLITE=1` / `FLASK_ENV=production` を含む）
4. `python init_db.py` でテーブルと管理者ユーザーを作成
5. Web タブで Manual configuration（WSGI から `create_app()` を呼ぶ）を設定して Reload

`app.py` はリバースプロキシ配下の HTTPS 判定に `ProxyFix` を適用しているため、PaaS 環境でも Secure Cookie が正しく機能します。

## セキュリティ設計

- **ログイン URL の隠蔽** — ログインページの URL を `ADMIN_LOGIN_PATH` で秘匿します。
- **ゲートキー方式** — 合言葉（`ADMIN_GATE_KEY`）を知る訪問者にのみログインページを表示し、それ以外には 404 を返してページの存在自体を隠します（フェイルクローズ設計）。
- **ブルートフォース対策** — ログイン失敗が続くと一定時間ロックアウトします。
- **パスワードのハッシュ化** — Werkzeug でハッシュ化して保存し、平文比較は行いません。
- **CSRF 保護** — Flask-WTF により全フォームに CSRF トークンを強制します。
- **アップロード検証** — 拡張子チェックに加えて先頭バイトから MIME タイプを判定し、拡張子偽装を検出します。ファイル名は UUID 化して保存します。
- **セキュリティヘッダー** — `X-Content-Type-Options` / `X-Frame-Options` / `Referrer-Policy` を全レスポンスに付与します。
- **未認証アクセスの偽装** — `@login_required` なページへ未ログインでアクセスすると、ログインへ誘導せず 404 を返します。
- **XSS 対策** — ユーザー入力（キャプション・地図の場所名など）は HTML 出力前にエスケープします。
