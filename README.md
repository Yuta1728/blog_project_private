# MITO Blog

Flask 製の個人ブログアプリケーションです。Markdown で記事を書き、画像・地図・YouTube を本文に埋め込めます。ジャンルとハッシュタグによる記事の分類・検索に対応し、管理画面は「秘密の URL ＋合言葉」で隠蔽するなど、個人運用に必要なセキュリティ対策を備えています。

## 主な機能

機能は「誰でも使える閲覧者向け機能」と「ログインした管理者だけが使える管理者向け機能」に分かれています。

### 閲覧者向けの機能

ログイン不要で、サイトを訪れた誰もが利用できる機能です。

- **記事一覧（トップページ）** — 新着順に記事を表示。サーバーサイドのページネーション（1 ページ 4 件）で、ページ番号による前後移動ができます。
- **キーワード検索** — 記事タイトルとハッシュタグ名を対象に部分一致で検索します。
- **ジャンル絞り込み** — ジャンル単位で記事を絞り込みます。トップの検索エリアでキーワードと同時に指定することも可能です。
- **ハッシュタグ絞り込み** — ジャンル選択中に、そのジャンル内で使われているタグを絞り込みバーから選べます。
- **ジャンル一覧ページ** — カテゴリ（ライフスタイル／娯楽など）ごとにアコーディオン形式でジャンルを整理して表示。ユーザーが独自作成したジャンルは「その他」に自動表示されます。
- **記事詳細ページ** — Markdown を HTML に変換して表示。目次（TOC）、本文中の画像とキャプション、Google マップ埋め込み、YouTube 埋め込み（サムネイルをタップして再生するファサード形式）に対応しています。
- **関連記事の表示** — 記事末尾に、同ジャンル×同タグ → 同タグ → 同ジャンル → 最新、の優先順位で最大 4 件の関連記事を表示します。
- **サイト統計** — トップページに総投稿数・使用ハッシュタグ数・最終更新日を表示します。
- **自己紹介ページ・使い方ページ** — 管理者プロフィールとサイトの使い方を案内する静的ページ。
- **ダークモード** — ヘッダーのボタンで明暗テーマを切り替え。選択は端末に保存され、次回訪問時も維持されます。
- **レスポンシブ対応** — スマートフォン向けのドロワーメニューやレイアウト最適化を備えています。

### 管理者向けの機能

ログインした管理者だけが利用できる機能です。単一管理者での運用を想定しています。

- **記事の投稿・編集・削除** — CSRF トークンで保護されたフォームから記事を管理します。
- **Markdown 編集ツールバー** — H2/H3 見出し・太字・目次・箇条書き・画像挿入・地図挿入・YouTube 挿入をワンクリックで本文に挿入。スマホでは編集時にツールバーがキーボード直上に固定されます。
- **画像アップロード** — 本文に載せる画像を複数枚アップロードできます（一括選択・1 枚ずつ追加の両対応）。各画像にキャプションを設定でき、拡張子・MIME タイプの二重検証と UUID によるファイル名ランダム化で安全に保存します。
- **サムネイル管理** — 一覧表示用のサムネイルを、専用画像のアップロード／プリセットからの選択／システム共通デフォルト、の優先順位で設定できます。
- **ハッシュタグ入力** — スペースやカンマ区切りでタグを入力でき、入力に応じてリアルタイムでタグのプレビューが表示されます。
- **ジャンルの新規作成** — 投稿フォームからその場で新しいジャンルを作成できます。
- **公開／非公開の切り替え** — 記事ごとに公開状態を切り替え。非公開記事は管理者本人だけが閲覧できます。
- **マイページ** — 自分の全投稿の一覧表示、使用ジャンルの確認、表示名（ニックネーム）の変更ができます。

## 技術スタック

| 分類 | 使用技術 |
|------|----------|
| 言語 | Python 3.10 |
| フレームワーク | Flask（Application Factory パターン） |
| ORM / マイグレーション | SQLAlchemy / Flask-Migrate（Alembic） |
| 認証 | Flask-Login |
| フォーム保護 | Flask-WTF（CSRF） |
| データベース | PostgreSQL（ローカル）／ SQLite（PythonAnywhere） |
| 本文変換 | Markdown（toc / nl2br 拡張） |
| ファイル検証 | filetype（MIME 判定） |
| フロントエンド | 素の HTML / CSS / JavaScript（Jinja2 テンプレート） |
| コンテナ | Docker Compose（ローカルの PostgreSQL） |

アプリは機能ごとに 3 つの Blueprint に分割されています。`auth`（ログイン・ログアウト）、`blog`（一般公開ページ）、`admin`（管理者専用ページ）です。

## ディレクトリ構成

```
.
├── app.py                 # エントリーポイント（create_app ファクトリ）
├── config.py              # .env の読み込み・設定値の提供
├── constants.py           # ジャンル定義（唯一の情報源）
├── extensions.py          # db / login_manager / migrate インスタンス
├── models.py              # ORM モデル（User / Post / Hashtag）
├── init_db.py             # SQLite 向け初期化スクリプト
├── views/
│   ├── auth.py            # ログイン・ログアウト
│   ├── blog.py            # 一覧・詳細・ジャンル・about・howto
│   └── admin.py           # 投稿・編集・削除・マイページ
├── templates/             # Jinja2 テンプレート
├── static/                # CSS / JS / 画像 / favicon
├── migrations/            # Alembic マイグレーション
├── docker-compose.yml     # ローカル用 PostgreSQL
├── requirements.txt       # 依存パッケージ（PostgreSQL 込み）
└── requirements-pythonanywhere.txt  # SQLite 運用向け依存
```

## セットアップ（ローカル開発）

ローカルでは Docker Compose の PostgreSQL に接続する構成です。

1. **リポジトリを取得して依存をインストール**

   ```bash
   python -m venv .venv
   source .venv/bin/activate      # Windows は .venv\Scripts\activate
   pip install -r requirements.txt
   ```

2. **`.env` を作成**（[環境変数](#環境変数)を参照）

3. **PostgreSQL を起動**

   ```bash
   docker compose up -d
   ```

4. **データベースを初期化**

   マイグレーションを適用します。

   ```bash
   flask db upgrade
   ```

   または、マイグレーションを使わずテーブル作成と管理者ユーザー登録を一括で行う場合:

   ```bash
   python init_db.py
   ```

5. **アプリを起動**

   ```bash
   python app.py
   ```

   ブラウザで `http://localhost:5000` を開きます。

## 環境変数

`.env` をプロジェクト直下に置きます（`config.py` が絶対パスで読み込みます）。

| 変数名 | 必須 | 説明 |
|--------|:---:|------|
| `SECRET_KEY` | 本番で必須 | セッション・CSRF トークンの署名鍵。長いランダム文字列を設定します。 |
| `ADMIN_USERNAME` | ✅ | 管理者ログインのユーザー名。 |
| `ADMIN_PASSWORD` | ✅ | 管理者パスワード（平文・ハッシュ済みのどちらでも可）。 |
| `ADMIN_LOGIN_PATH` | ✅ | ログインページの URL パス（推測されにくい文字列）。 |
| `ADMIN_GATE_KEY` | ✅ | ログインページを表示するための合言葉。 |
| `FLASK_ENV` | 任意 | `production` にすると本番モード（Secure Cookie・鍵の必須化）。 |
| `USE_SQLITE` | 任意 | `1` で SQLite（`instance/blog.db`）を使用。 |
| `DATABASE_URL` | 任意 | 指定時は最優先で接続先に使用。 |
| `POSTGRES_USER` / `POSTGRES_PASSWORD` / `POSTGRES_DB` | 任意 | ローカルの PostgreSQL 接続情報。 |

ランダム文字列は次のように生成できます。

```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

**DB 接続先の優先順位**は `DATABASE_URL` → `USE_SQLITE=1`（SQLite）→ ローカル PostgreSQL の順です。

## 記事本文で使える独自記法

Markdown に加えて、以下の独自タグを本文で使えます（編集ツールバーからも挿入可能）。

| 記法 | 効果 |
|------|------|
| `[toc]` | その位置に目次を展開します。 |
| `[img1]`, `[img2]` … | アップロードした N 番目の画像を挿入します。 |
| `[map:場所名]` | Google マップを埋め込みます（例: `[map:東京スカイツリー]`）。 |
| `[youtube:URL]` | YouTube 動画を埋め込みます。通常 URL・短縮 URL・Shorts・埋め込み URL・動画 ID に対応。 |

2 行以上の連続した空行は行間として保持され、意図的に余白を空けられます。

## セキュリティ設計

個人運用の管理画面を守るため、多層的な対策を実装しています。

- **管理画面の隠蔽** — ログイン URL を環境変数で秘匿し、さらに「合言葉（ゲートキー）」の Cookie を持たない訪問者にはログインページの存在自体を 404 で隠します。合言葉が未設定の場合は無条件に 404 とするフェイルクローズ設計です。
- **ブルートフォース対策** — ログインに一定回数失敗するとセッション単位で一時ロックアウトします。
- **パスワードのハッシュ化** — 平文は保存せず、ハッシュ値と照合します。
- **CSRF 保護** — 全フォームに CSRF トークンを強制します。
- **ファイルアップロード検証** — 拡張子チェック・MIME タイプ判定・UUID 命名・容量制限（30MB）の多層防御。保存はアトミックに行い、途中失敗時の孤立ファイルを残しません。
- **出力エスケープ** — 画像キャプション・地図・YouTube タグなど、HTML に直接組み立てる箇所を確実にエスケープして XSS を防ぎます。
- **Open Redirect 対策** — リダイレクト先が同一オリジンかを検証します。
- **セキュリティヘッダー** — `X-Content-Type-Options` / `X-Frame-Options` / `Referrer-Policy` を付与します。
- **リバースプロキシ対応** — ProxyFix と Secure Cookie により、本番の HTTPS 環境でも正しく動作します。

未ログインで管理ページにアクセスされた場合も、ログイン画面へ誘導せず 404 を返して存在を隠します。

## デプロイ

- **PythonAnywhere（無料枠）** — SQLite で運用する手順を `deploy_pythonanywhere.md` に詳しくまとめています。`USE_SQLITE=1` と `FLASK_ENV=production` を設定し、`init_db.py` で初期化してから WSGI を設定します。
- **その他の PaaS** — `DATABASE_URL` を設定すれば PostgreSQL 等に接続できます。`SECRET_KEY` の設定を忘れると本番モードでは起動時にエラーになります。

WSGI 設定のサンプルは `wsgi_pythonanywhere.py` を参照してください。
