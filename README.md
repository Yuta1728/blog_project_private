# MITO Blog

Flask 製の個人用ブログアプリケーションです。記事の投稿・管理を行う管理者機能と、記事を閲覧・検索する公開機能を備え、Markdown による執筆、画像の自動最適化、地図・YouTube 埋め込み、ダークモードなどに対応しています。

**Application Factory パターン**（`create_app()`）と **Blueprint** による機能分割で構成され、DB は **SQLite / PostgreSQL の両対応**です。

## 主な機能

機能を「誰でも使える閲覧者向け」と「ログインが必要な管理者向け」に分けて整理します。

### 閲覧者向け機能

ログイン不要で、誰でも利用できる公開ページの機能です。

| 機能 | 説明 |
|------|------|
| **記事一覧（トップページ）** | 公開記事を新しい順に表示。サーバーサイドのページ番号方式で 1 ページ 4 件ずつ表示します。 |
| **キーワード検索** | タイトルまたはハッシュタグ名の部分一致で記事を検索できます（大文字小文字を区別しません）。 |
| **ジャンル絞り込み** | ジャンル別に記事を絞り込めます。ジャンル一覧ページはカテゴリごとのアコーディオン表示です。 |
| **ハッシュタグ絞り込み** | ジャンル選択中に、そのジャンル内で使われているタグで、さらに絞り込みできます。 |
| **記事詳細ページ** | Markdown で書かれた本文を HTML に変換して表示。目次・画像キャプション・地図・動画埋め込みに対応します。 |
| **地図埋め込み** | 本文に埋め込まれた Google マップをその場で確認できます。 |
| **YouTube 埋め込み** | サムネイル画像をタップするとその場で動画再生が始まる軽量な「ファサード」方式です。 |
| **関連記事表示** | 記事末尾に、同ジャンル・同タグなどを基準にした関連記事を最大 4 件表示します。 |
| **統計情報** | トップページに総投稿数・使用ハッシュタグ数・最終更新日を表示します。 |
| **ダークモード** | ヘッダーのボタンで明暗テーマを切り替え。選択内容はブラウザに保存され、次回訪問時も維持されます。 |
| **レスポンシブ対応** | PC・スマホ両対応。スマホでは専用のドロワーメニューやスクロール連動ヘッダーが動作します。 |
| **自己紹介・使い方ページ** | 管理者プロフィールやブログの使い方を紹介する静的ページを備えます。 |

### 管理者向け機能

隠された URL からログインした管理者のみが利用できる機能です。

| 機能 | 説明 |
|------|------|
| **記事の投稿・編集・削除** | Markdown で記事を執筆し、公開／非公開を切り替えて投稿できます。編集・削除も可能です。 |
| **Markdown 編集ツールバー** | H2/H3 見出し・太字・目次・箇条書きリスト・地図・YouTube・画像挿入をボタンで挿入できます。スマホではキーボード直上にツールバーが固定され、カーソル位置も自動追従します。 |
| **画像アップロードと自動最適化** | 本文画像を複数枚アップロードでき、`[imgN]` タグで本文中に挿入します。アップロード時に Pillow で長辺 1600px まで縮小・再圧縮します。 |
| **画像キャプション** | 各画像に説明文（キャプション）を付けられます。 |
| **サムネイル画像の個別設定** | 記事一覧用のサムネイルを、専用アップロード・プリセット選択・システム既定の 3 段階の優先順位で設定できます。専用画像は幅 400px の WebP に自動変換されます。 |
| **ハッシュタグ管理** | スペース・カンマ区切りでタグを入力でき、リアルタイムプレビュー付き。どの記事にも属さなくなった孤立タグは自動削除されます。 |
| **ジャンル管理** | プリセットのジャンルから選択、または新規ジャンルを作成できます。 |
| **公開設定** | 記事ごとに全体公開／非公開をトグルで切り替え可能。非公開記事は管理者本人のみ閲覧できます。 |
| **マイページ** | 自分の投稿一覧・使用ジャンル・総投稿数を確認でき、ニックネームを変更できます。 |

## 技術スタック

| 分類 | 使用技術 |
|------|----------|
| 言語 | Python 3.10 |
| Web フレームワーク | Flask 3.x（Application Factory + Blueprint） |
| ORM / DB | SQLAlchemy 2.x、Flask-SQLAlchemy、Flask-Migrate（Alembic） |
| データベース | SQLite / PostgreSQL（psycopg 3） |
| 認証 | Flask-Login |
| フォーム / CSRF | Flask-WTF（CSRFProtect） |
| 画像処理 | Pillow（縮小・WebP 変換）、filetype（MIME 検証） |
| Markdown | Markdown（toc / nl2br 拡張） |
| フロントエンド | 素の HTML / CSS / JavaScript（フレームワーク不使用） |

## ディレクトリ構成

```
.
├── app.py                  # エントリーポイント（create_app ファクトリ）
├── config.py               # .env からの環境変数読み込み
├── constants.py            # ジャンル定義（唯一の情報源）
├── extensions.py           # db / login_manager / migrate インスタンス
├── models.py               # User / Post / Hashtag モデル
├── init_db.py              # SQLite 用の初期化スクリプト
├── views/
│   ├── auth.py             # ログイン・ログアウト
│   ├── blog.py             # 公開ページ（一覧・詳細・ジャンル）
│   └── admin.py            # 管理ページ（投稿・編集・削除・マイページ）
├── templates/              # Jinja2 テンプレート
├── static/
│   ├── css/                # ページ別スタイル + ダークモード
│   ├── js/editor.js        # 投稿・編集エディタの共通ロジック
│   └── img/                # 記事画像・サムネイル
├── migrations/             # Alembic マイグレーション
├── docker-compose.yml      # ローカル開発用 PostgreSQL
├── requirements.txt        # 依存パッケージ（PostgreSQL 含む）
└── requirements-pythonanywhere.txt  # SQLite 運用向け依存
```

## セットアップ（ローカル開発）

### 1. リポジトリの取得と仮想環境の作成

```bash
git clone <このリポジトリの URL>
cd <プロジェクトディレクトリ>

python -m venv venv
source venv/bin/activate        # Windows は venv\Scripts\activate

pip install -r requirements.txt
```

### 2. 環境変数ファイル（.env）の作成

プロジェクト直下に `.env` を作成します（詳細は[環境変数](#環境変数)を参照）。

### 3. データベースの用意

**PostgreSQL を使う場合**（`docker-compose.yml` 利用）:

```bash
docker compose up -d          # localhost:15432 で PostgreSQL が起動
flask db upgrade              # マイグレーションを適用
```

**SQLite を手軽に使う場合**:

`.env` に `USE_SQLITE=1` を設定してから、

```bash
python init_db.py             # テーブル作成 + 管理者ユーザー作成
```

### 4. 起動

```bash
python app.py
```

ブラウザで `http://localhost:5000` にアクセスします。

## 環境変数

`.env` ファイル、または OS の環境変数として設定します。

| キー | 必須 | 説明 |
|------|:----:|------|
| `SECRET_KEY` | 本番で必須 | セッション・CSRF トークンの署名鍵。長いランダム文字列。 |
| `ADMIN_USERNAME` | ✅ | 管理者ログインのユーザー名。 |
| `ADMIN_PASSWORD` | ✅ | 管理者パスワード（平文・ハッシュ済みどちらも可）。 |
| `ADMIN_LOGIN_PATH` | ✅ | ログインページの URL パス（推測されにくい文字列）。 |
| `ADMIN_GATE_KEY` | ✅ | ログイン画面を表示するための合言葉。 |
| `FLASK_ENV` | 任意 | `production` で本番モード（Secure Cookie 有効化など）。 |
| `USE_SQLITE` | 任意 | `1` で SQLite（`instance/blog.db`）を使用。 |
| `DATABASE_URL` | 任意 | 明示的な DB 接続 URL（最優先）。 |
| `POSTGRES_USER` / `POSTGRES_PASSWORD` / `POSTGRES_DB` | 任意 | ローカルの PostgreSQL 接続情報。 |

ランダム文字列の生成例:

```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

> **DB 接続の優先順位**: `DATABASE_URL` → `USE_SQLITE=1` → ローカル PostgreSQL（docker-compose）の順に判定されます。

## データベースの初期化・マイグレーション

**Alembic を使う場合**（PostgreSQL 運用推奨）:

```bash
flask db upgrade              # 最新スキーマまで適用
flask db migrate -m "変更内容" # models.py 変更時に差分を生成
```

**init_db.py を使う場合**（SQLite 運用）:

```bash
python init_db.py
```

`db.create_all()` でテーブルを作成し、Alembic の履歴を head にスタンプするため、後からマイグレーション運用へ移行してもスキーマの不整合が起きません。管理者ユーザーも同時に作成されます（何度実行しても安全）。

## セキュリティ設計

このアプリは学習目的ながら、実運用を意識した多層的なセキュリティ対策を備えています。

- **ログイン URL の隠蔽**: ログインページの URL は環境変数で設定する秘密のパスです。
- **ゲートキー方式**: 合言葉（`?key=`）または専用 Cookie を持たない訪問者には 404 を返し、ログインページの存在自体を隠します。設定漏れ時は無防備にならないよう常に 404 とする「フェイルクローズ」設計です。
- **ブルートフォース対策**: 連続 5 回のログイン失敗で 5 分間ロックアウトします。
- **パスワードのハッシュ化**: パスワードは Werkzeug でハッシュ化して保存し、平文比較は行いません。
- **CSRF 保護**: 全フォームに CSRF トークンを強制適用します。
- **ファイルアップロード検証**: 拡張子・MIME タイプの多層チェックと UUID によるファイル名ランダム化、30MB の容量制限を行います。
- **XSS 対策**: 画像キャプションや地図・動画タグなど、HTML に直接埋め込む箇所は出力時にエスケープします。
- **Open Redirect 対策**: リダイレクト先が同一オリジンかを検証します。
- **セキュリティヘッダー**: `X-Content-Type-Options`、`X-Frame-Options`、`Referrer-Policy` を付与します。
- **プロキシ対応**: `ProxyFix` によりリバースプロキシ配下でも HTTPS を正しく判定します。

## デプロイ

**PythonAnywhere 無料枠（SQLite）** へのデプロイ手順は、`deploy_pythonanywhere.md` に詳しくまとめています。

概要:

1. GitHub からリポジトリを `git clone`
2. 仮想環境を作成し `requirements-pythonanywhere.txt` をインストール
3. サーバー上に `.env` を作成（`USE_SQLITE=1` / `FLASK_ENV=production` を含む）
4. `python init_db.py` で DB を初期化
5. Web タブで Manual configuration を設定し、WSGI ファイルで `create_app()` を公開
6. Reload して公開

更新は「PC で修正 → GitHub に push → サーバーで pull → Reload」が基本の流れです。
